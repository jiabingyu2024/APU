# PYNQ-Z2 AI-Readable Board Reference

This file is a structured conversion of the PYNQ-Z2 Reference Manual v1.0. It is
organized for machine reading and for assisting with Vivado and XDC work. Values
marked as a warning or erratum must be checked against the official schematic or
master XDC before hardware use.

## 1. Critical Rules for Vivado and XDC

1. The target device is `xc7z020clg400-1`.
2. `PL pin`, `FPGA pin`, and `ZYNQ PIN` in the tables below mean Zynq package
   pins usable with XDC `PACKAGE_PIN` constraints.
3. `MIO[n]` belongs to the Processing System (PS). Do not constrain PS MIO with
   normal PL `PACKAGE_PIN` lines. Configure it in the Zynq7 Processing System IP.
4. Most ordinary PL GPIO on this board is 3.3 V logic. A typical constraint uses
   `IOSTANDARD LVCMOS33`, but the official master XDC and schematic remain the
   authority.
5. HDMI data and clock pairs are differential signals. Do not treat them as
   unrelated single-ended GPIO.
6. XADC dedicated analog pins and analog-capable Arduino pins need XADC-aware
   handling. Do not blindly apply ordinary digital I/O constraints.
7. Pmod A signals are shared with Raspberry Pi header pins. Do not drive both
   connector names as independent interfaces.
8. The manual does not provide package-pin mappings for every board peripheral,
   notably the ADAU1761 audio codec. Use the official schematic/master XDC for
   omitted signals.

## 4. Power, Boot, and Reset

### 4.1 Power

- Set jumper `J9` to `USB` for Micro-USB power through `J8`.
- Set `J9` to `REG` for the external barrel input or Arduino-header battery input.
- External barrel input `DC1`: 7-15 V DC, center-positive, 2.1 mm inner diameter;
  12 V is recommended.
- A battery positive terminal may connect to Arduino `J7 VIN`; negative connects
  to `GND`; `J9` must be set to `REG`.
- Pmod VCC/GND pins can deliver up to 1 A according to the manual. Board-level
  supply and thermal limits still apply.

### 4.2 Boot

Boot source is selected by jumper `JP1`:

- MicroSD
- Quad SPI Flash
- JTAG

### 4.3 Reset

- `SRST`: system reset for PS and PL; clears PS memory and PL configuration, but
  does not re-sample boot-mode straps.
- `PROG`: asserts Zynq `PROG_B`, clears the PL configuration, and deasserts DONE.
- The regulator `PGOOD` signal holds the device in reset until supplies are valid.
- `SRST` also toggles the Arduino shield reset signal `CK_RST`.

## 5. Clock Sources

| Clock | Destination | Frequency | Pin/configuration | Notes |
|---|---|---:|---|---|
| PS clock | Zynq `PS_CLK` | 50 MHz | PS dedicated clock | Used to generate PS subsystem clocks |
| PL reference clock | PL | 125 MHz | package pin `H16` | Generated by Ethernet PHY reference output |

Important: asserting Ethernet PHY reset `PHYRSTB` low disables the PHY and its
`CLK125` output. A PL design relying on package pin `H16` therefore also depends on
the Ethernet PHY being out of reset.

Example PL clock constraint, subject to official XDC verification:

```tcl
set_property PACKAGE_PIN H16 [get_ports clk_125mhz]
set_property IOSTANDARD LVCMOS33 [get_ports clk_125mhz]
create_clock -name clk_125mhz -period 8.000 [get_ports clk_125mhz]
```

## 6. PS-Connected Peripherals and MIO Maps

These signals are configured in the Zynq Processing System IP, not as ordinary PL
ports in an XDC file.

### 6.1 Quad SPI Flash

Device: Spansion `S25FL128S`, 16 MB, x1/x2/x4, up to 104 MHz bus clock.

| MIO | Signal |
|---:|---|
| 1 | CS |
| 2 | DQ0 |
| 3 | DQ1 |
| 4 | DQ2 |
| 5 | DQ3 |
| 6 | SCLK |
| 7 | VCFG0 |
| 8 | SCLK feedback (`qspi_sclk_fb_out`) |

The QSPI OTP address range `0x20` through `0x25` stores a factory-programmed
48-bit globally unique EUI-48/64-compatible identifier.

### 6.2 USB Host ULPI

The TI TUSB1210 PHY connects to PS USB0 and supports USB 2.0 host mode up to
480 Mb/s. USB OTG and USB device modes are not supported by this board wiring.

| 28 | DATA4 | 35 | DATA3 |

| 30 | STP | 37 | DATA5 |

### 6.3 MicroSD / SDIO0

### 7.5 HDMI Ports

Both HDMI ports connect directly to the PL and are electrically unbuffered. The
board labels one as source/output (`J11`) and one as sink/input (`J10`), but the PL
designer controls direction. The manual's table heading contains an unexplained
extra `J18` token; treat `J11` and `J10` as the connector references and verify the
schematic.

| HDMI signal | J11 source/output package pins | J10 sink/input package pins |
|---|---|---|
| D2_P, D2_N | J18, H18 | N20, P20 |
| D1_P, D1_N | K19, J19 | T20, U20 |
| D0_P, D0_N | K17, K18 | V20, W20 |
| CLK_P, CLK_N | L16, L17 | N18, P19 |
| CEC | G15 | H17 |
| DDC SCL, SDA | B9, B13 | U14, U15 |
| HPD/HPA | R19, hot-plug detect input, inverted | T19, hot-plug assert output |

HDMI constraints normally require differential-pair-aware port naming and an
appropriate differential I/O standard such as the value used by the official master
XDC. Do not infer polarity only from package-pin ordering; preserve `_P` and `_N`.

### 7.12 Arduino Analog-Capable Header J1

The `A0-A5` connector inputs are scaled by an external resistor network so that
0-3.3 V at the header becomes a valid 0-1 V XADC input. They also connect to PL
before the divider and may be used as digital I/O. Digital use still requires care
because the analog network remains attached.

| J1 pin | Signal | Package pin | Header analog range | Digital capable |
|---:|---|---|---|---|
| 1 | A5 | U10 | 0-3.3 V | Yes |
| 2 | A4 | T5 | 0-3.3 V | Yes |
| 3 | A3 | V11 | 0-3.3 V | Yes |
| 4 | A2 | W11 | 0-3.3 V | Yes |
| 5 | A1 | Y12 | 0-3.3 V | Yes |
| 6 | A0 | Y11 | 0-3.3 V | Yes |

Arduino digital-pin voltage limits from the manual:

| Board state | Absolute minimum | Recommended minimum | Recommended maximum | Absolute maximum |
|---|---:|---:|---:|---:|
| Powered | -0.4 V | -0.2 V | 3.4 V | 3.75 V |
| Unpowered | -0.4 V | N/A | N/A | 0.55 V |

The board is not compatible with shields that output 5 V digital or analog signals.
The manual warns that excessive voltage may damage the Zynq.

### 7.13 Dedicated XADC Header J5

| J5 pin | Signal | Description | Zynq connection |

## 8. XDC Construction Patterns

Use RTL top-level port names that match your design. The names below are examples,
not mandatory board-file names.

### 8.1 Buttons, Switches, and LEDs

```tcl
set_property -dict {PACKAGE_PIN D19 IOSTANDARD LVCMOS33} [get_ports {btn[0]}]
set_property -dict {PACKAGE_PIN D20 IOSTANDARD LVCMOS33} [get_ports {btn[1]}]
set_property -dict {PACKAGE_PIN L20 IOSTANDARD LVCMOS33} [get_ports {btn[2]}]
set_property -dict {PACKAGE_PIN L19 IOSTANDARD LVCMOS33} [get_ports {btn[3]}]

set_property -dict {PACKAGE_PIN M20 IOSTANDARD LVCMOS33} [get_ports {sw[0]}]
set_property -dict {PACKAGE_PIN M19 IOSTANDARD LVCMOS33} [get_ports {sw[1]}]

set_property -dict {PACKAGE_PIN R14 IOSTANDARD LVCMOS33} [get_ports {led[0]}]
set_property -dict {PACKAGE_PIN P14 IOSTANDARD LVCMOS33} [get_ports {led[1]}]
set_property -dict {PACKAGE_PIN N16 IOSTANDARD LVCMOS33} [get_ports {led[2]}]
set_property -dict {PACKAGE_PIN M14 IOSTANDARD LVCMOS33} [get_ports {led[3]}]
```

### 8.2 Generic Expansion GPIO

```tcl
# Example: Pmod A pin 1
set_property PACKAGE_PIN Y18 [get_ports pmod_a_1]
set_property IOSTANDARD LVCMOS33 [get_ports pmod_a_1]

# Example: Raspberry Pi header pin 7
set_property PACKAGE_PIN V6 [get_ports rpi_pin7]
set_property IOSTANDARD LVCMOS33 [get_ports rpi_pin7]
```

### 8.3 Basic Validation Commands

Run these in the Vivado Tcl console after synthesis or implementation:

```tcl
report_io
report_drc
report_clock_networks
report_methodology
```

Useful checks before generating a bitstream:

- Every external RTL port that reaches a package pin has the intended
  `PACKAGE_PIN` and `IOSTANDARD`.
- No two unrelated top-level ports claim the same package pin.
- Shared Pmod A/Raspberry Pi pins are represented only once electrically.
- Input/output direction agrees with the connected peripheral.
- Active-low RGB LED behavior is handled in RTL.
- Differential HDMI P/N pairs and clock constraints are preserved.
- PS MIO peripherals are enabled and mapped in the Zynq PS configuration, not
  duplicated as PL ports.
- The 125 MHz PL clock is available only when the Ethernet PHY is not held reset.
